1. Introduction

1.1 Purpose of Product

The purpose of this project is to develop a Unity based multiplayer game incorporating two separate roles: host and audience. The host draws an object, while the audience uses a chat box to deduce what the drawing depicts. The game is targeted towards young adults and aims to challenge players' abilities in object recognition and artistic ability while having fun competing online. 

1.2 Scope of Product

Our proposed game includes several key features, including an online lobby where players can either join or host a game, a drawing board with a pen and eraser tool for the host and a chatbox for players to guess the drawing and possibly engage in conversation during gameplay. To keep the gameplay competitive a countdown timer and a point system will be integral. Additionally the game will provide a word bank of drawing prompts as well as a reporting mechanism for uncooperative or abusive players.

The proposed game does not include artificial intelligence (AI) for either the host or the audience to generate or guess drawings. Communication in our game will be limited to text based chat discluding voice chat, emojis and file uploads.

1.3 Acronyms, Abbreviations, Definitions

  1. Unity: A cross-platform game engine used for building real-time 3D and 2D assets/projects across many different industries.
  2. Multiplayer: Refers to games that are played by more than one person simultaneously.
  3. Chatbox: Text based communication for multiple players.
  4. Word bank: A set of words that are pre selected for the host to choose from.
  5. Drawer: The initial host of the game and also the singular role of drawing objects for the audience to guess.
  6. Audience: The users who join a hosted game and guess what is being drawn by the host.

2. General Description of Product

2.1 Context of Product

Our game will be a Unity based Chrome and Firefox compatible multiplayer game hosted on simmer.io. The game will allow users to host a game or join a game that is already being hosted. The host of the game serves as the drawer and the other users as the audience, as the game progresses the role of the drawer cycles through the audience in the order that they joined the game. If a host leaves before the game is finished or is booted due to complaints the next host is assigned in the same manner. 

Each game will be timed and points will be awarded based on correctly guessing the drawing in a timely manner. Players in the audience will also be subject to the same reporting mechanism as the host. 

2.2 Domain Model

Our domain model will consist of a player class defined as host or audience, game server, the canvas or drawing board, the chat system and the scoring system. The player class will have attributes, name, email, score and password. The host for the game shares a server for other players to join. The drawing board class will contain three attributes,  title, difficulty and image. The image attribute will store a png of the canvas.

The game server class will consist of players, status of players, and the canvas. The status is used to indicate whether the game room is currently waiting for players, in progress, or finished. The canvas attribute is a reference to the drawing that is currently being guessed. The score class contains an integer based score attribute to represent a score achieved by players and a reference to each player.

2.3 Product Functions

Some of the general product functions include:

  1. Drawing creation:

    The game enables players to create their own drawings using a variety of tools and features. Players can use different colors, brushes, and patterns to create unique and complex drawings.

  2. Guessing game:

    The game also involves guessing the drawings of other players. Players are presented with a drawing and are required to guess what it represents. The game keeps track of the number of correct guesses made by each player.

  3. Multiplayer:

    "Guess That Drawing" is a multiplayer game, allowing players to compete against each other in real-time. Players can invite their friends to play or join public games.

  4. Chat:

    The game also features a chat function, allowing players to communicate with each other during the game. This enables players to collaborate, share tips and hints, or just chat with each other.

  5. Scoring system:

    The game features a scoring system that rewards players for correct guesses and successful drawings. Points earned can be used to unlock new features or levels in the game.

2.5 Constraints:

There are several constraints on the "Guess That Drawing" system:

  1. Hardware constraints:

    The game's performance may be limited by the hardware specifications of the devices used to run it. Too old or less powerful devices may not be able to handle the game's graphics and processing demands.

  2. Bandwidth limitations:

    The game requires an internet connection to function, and the performance of the game may be impacted by limited bandwidth. This could lead to slower loading times, lagging, or poor video quality.

  3. Time and resource constraints:

    Developing a high-quality online game can be time-consuming and require significant resources, including human and financial resources. These constraints could impact the overall quality of the game and the speed at which new features are added.

  4. User experience:

    The success of the game will depend on the user experience. Constraints on the user interface, user interaction, and usability can impact the overall satisfaction and enjoyment of players. Therefore, the design and development of the game should take into account user feedback and user testing to ensure a positive user experience.
